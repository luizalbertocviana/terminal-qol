✓ Bash 4+ ✓ macOS + Linux ✓ ShellCheck Clean ✓ Idempotent ✓ Zero Network on Load

Installation
1
Download the script

Place it anywhere on your system. Recommended location:

mkdir -p ~/.config/qol
cp terminal-qol.bash ~/.config/qol/
2
Source from ~/.bashrc

Add this single line at the end of your ~/.bashrc:

# terminal-qol — quality of life enhancements
[ -f "$HOME/.config/qol/terminal-qol.bash" ] && \
  source "$HOME/.config/qol/terminal-qol.bash"
3
Optional: Pre-configure before sourcing

Set config variables before the source line to customize behavior:

# Disable features you don't want
export QOL_ENABLE_K8S=0
export QOL_SAFE_ALIASES=0  # if you prefer unguarded rm
export QOL_ENABLE_PROMPT=0 # if you manage your own prompt
[ -f "$HOME/.config/qol/terminal-qol.bash" ] && \
  source "$HOME/.config/qol/terminal-qol.bash"
4
Reload your shell
source ~/.bashrc  # or open a new terminal

Then run qol_doctor to see what's active, and qol_help for the full reference.

Design Choices

Idempotency via Guard Variable

The __QOL_LOADED guard ensures re-sourcing the script (e.g. via reload_shell) doesn't duplicate PROMPT_COMMAND entries, PATH entries, or trap registrations. Functions are redefined on each source, which is harmless and expected.

Feature Detection, Not Feature Assumption

Every external tool is gated behind has_cmd. The script never assumes a tool is present because it was present when the script was written. This makes it safe on minimal containers, fresh VMs, and developer machines alike.

One Warning Per Tool Per Session

The __QOL_WARNED associative array tracks which warnings have fired. A missing tool emits one warning at source time, not every time you invoke something that needs it. This keeps shells usable without spamming every prompt.

No eval Except Shell Integrations

eval is used only for starship init bash, zoxide init bash, and direnv hook bash — these tools output Bash code designed to be eval'd. There is no user-data eval path. The envload function uses regex-validated assignment rather than eval to load .env files.

Safe .env Loading

envload parses .env files line-by-line using Bash regex, not eval or source. It validates that each line is of the form VALID_KEY=value before exporting, skips comments and blank lines, and strips surrounding quotes. Malformed lines are warned about, not silently dropped or executed.

Startup Performance

The script uses only built-in Bash features for the core setup. External commands are called only when they're needed (detection via command -v, which is a shell builtin). Shell integrations like eval "$(zoxide init bash)" do incur a subprocess but only when those tools are present and enabled.

Portability Notes

Some features require Bash 4+ (associative arrays, globstar, autocd). macOS ships with Bash 3.2 by default — users should install a modern Bash via Homebrew. The script degrades gracefully on Bash 3.2 but emits a version warning. The EPOCHREALTIME variable for load timing requires Bash 5+.

Namespacing Strategy

Internal functions and variables are prefixed with __qol_ or __QOL_. User-facing commands use short, memorable names. Feature-group state is stored in __QOL_FEATURES (an associative array) rather than a pile of separate variables, making qol_doctor easy to implement generically.
